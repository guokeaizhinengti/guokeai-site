## 先确定“国内”的含义

用户说“国内上线”可能指：

1. 服务器物理位置在中国大陆境内；
2. 用户主要在中国大陆，但服务器在香港或境外；
3. 只要求中文域名/中文用户体验；
4. 要求特定国内云厂商。

这四种路线的备案、性能、网络和供应商步骤不同。先确认服务器地域和目标用户，不要仅凭域名后缀判断。

## 中国大陆境内服务器

把合规作为架构前置条件，不要等技术部署结束后才发现无法公开接入。

- 核验域名实名、接入服务商、主体类型、服务性质和当前 ICP 备案或经营许可要求。
- 非经营性与经营性互联网信息服务的要求不同；特定内容领域还可能需要前置审批。
- 按接入服务商的备案系统提交，并以工信部及属地通信管理局当前口径为准。
- 核验备案编号的页面展示和链接要求。
- 继续核验公安联网备案、隐私、数据、内容、算法或行业许可是否适用于具体业务；不要把 ICP 备案当作所有合规的替代品。
- 不提供法律结论；识别风险并引导用户查看官方办理渠道或咨询专业人士。

## 中国大陆以外服务器

- 不要把“无需走大陆境内接入备案流程”简化成“没有任何合规要求”。
- 从中国大陆不同运营商和地区测试时延、丢包、DNS 和 TLS。
- 检查页面字体、脚本、图片、分析工具、验证码、地图、支付和 API 是否依赖境外资源。
- 评估跨境链路抖动和供应商服务条款，准备可替换依赖。
- 如果未来迁入大陆境内，把备案和接入迁移作为单独项目规划。

## 国内网络依赖

在服务器上实际测试：

```bash
git ls-remote <REPOSITORY_URL>
curl -I --max-time 15 <PACKAGE_REGISTRY_OR_API>
```

若 GitHub 或包源不稳定：

1. 优先由 CI 构建带校验和的发布制品。
2. 使用官方或可信的国内镜像，并锁定依赖与校验完整性。
3. 使用对象存储或制品仓库传递版本化产物。
4. 保留 GitHub 为版本事实源，记录制品与提交 SHA 的对应关系。

不要关闭证书校验，不要使用未知公共代理，不要让服务器 Deploy Key 获得不必要的写权限。

## DNS 与端口

- A 记录指向 IPv4；AAAA 指向 IPv6；CNAME 指向另一个主机名。
- “主机记录”通常只填子域前缀，但各供应商用词不同，以当前控制台说明为准。
- `0.0.0.0/0` 表示所有 IPv4 来源，不是一个服务器地址；只对 80/443 这样确需公开的端口使用。
- IPv6 公网开放是独立决策，通常还要单独处理 `::/0`、AAAA 和防火墙。
- 云安全组、系统防火墙、容器端口映射和进程监听地址是不同层。
- DNS TTL 只影响缓存更新，不会修复错误 IP、关闭端口或证书不匹配。

## 官方核验原则

规则会变更。执行时优先检索并引用：

- 工业和信息化部及属地通信管理局
- 工信部备案管理系统
- 实际接入服务商的备案与域名文档
- 云厂商的安全组、DNS 和证书官方文档

截至创建本技能时，可用于定位官方规则的入口包括：

- 工信部《互联网信息服务管理办法》
- 工信部《非经营性互联网信息服务备案管理办法》
- 工信部备案管理系统 `https://beian.miit.gov.cn/`

把这些入口当作核验起点，不把技能内文字当作最新法律意见。
