## 排障纪律

每次记录：

```text
测试：
结果：
证明了什么：
尚未证明什么：
下一步唯一动作：
```

先测试最内层，再向外移动。一次只改一个变量，改后立即复测。

## 症状矩阵

| 症状 | 最小测试 | 常见推断 | 下一层动作 |
|---|---|---|---|
| 构建失败 | 在项目目录运行生产构建 | 依赖、版本、类型或环境缺失 | 核对锁文件、运行时版本和完整错误 |
| 本地构建成功，平台构建失败 | 对比平台根目录、运行时、命令和变量 | monorepo 路径、版本或构建变量不一致 | 只修一个配置并重新部署目标 SHA |
| 预览正常，生产功能失败 | 对比预览/生产变量、域名和数据源 | 环境作用域或生产回调未配置 | 核对变量名称，不输出变量值 |
| 平台部署成功但页面 404 | 查输出目录、路由与部署日志 | 产物目录或 SPA/SSR 路由规则错误 | 核对框架产物与平台路由配置 |
| 平台函数超时或休眠 | 查函数日志、时长、区域和生命周期 | 工作负载不适合当前函数模型 | 优化任务或改用常驻应用平台/VPS |
| 进程启动即退出 | 查看服务日志与退出码 | 环境变量、端口、路径或权限错误 | 用服务用户和相同环境直接验证启动 |
| `127.0.0.1:<port>` 拒绝连接 | `ss -lntp` 与本机 health | 没监听或监听端口不同 | 修进程，不碰 DNS/Nginx |
| 本机应用通，Nginx 502 | Nginx error log、proxy 目标 | 上游地址/路径/权限错误 | 核对回环地址、端口和 `proxy_pass` |
| 首页通，刷新子路由 404 | 直接请求 SPA 深链接 | 缺少 `index.html` 回退 | 配置 `try_files` 或框架等价规则 |
| 服务器本机 Nginx 通，公网 IP 超时 | 从外部请求 80；查安全组与系统防火墙 | 公网入口未放行或无公网 IP | 分别验证云安全组、系统防火墙、路由 |
| 公网 IP 通，域名解析不到 | `dig/Resolve-DnsName` | DNS 记录、权威服务器或 TTL 问题 | 查权威记录并逐字符核对域名/IP |
| 域名指向旧 IP | 查询多个公共 DNS | TTL 缓存或修改了错误 DNS 区域 | 核对域名 NS、等待 TTL、移除冲突记录 |
| HTTP 通，申请证书失败 | 检查 80、DNS、ACME 日志 | 域名控制验证未通过或频率限制 | 修前置条件后重试，避免盲目重复申请 |
| HTTPS 证书不匹配 | 查看证书 SAN 与 SNI | 请求落到错误 server block 或证书不含域名 | 核对 `server_name`、证书路径和 DNS |
| 平台自定义域名一直待验证 | 查平台要求的当前 DNS 记录与权威解析 | 记录复制错误、冲突记录或修改了错误 DNS 区域 | 以当前项目页面的记录为准逐项核对 |
| 首页通，API 404 | 直接请求 `/api` 与上游路由 | 前缀被重复或剥离 | 检查 `location` 和 `proxy_pass` 尾斜杠 |
| API 返回客户端 IP/限流异常 | 查转发头和应用代理设置 | trust proxy 层数不正确 | 只信任明确入口代理并复测 |
| 服务器拉不到 GitHub | `ssh -vT`、`git ls-remote` | 密钥未选中、Deploy Key 冲突或网络问题 | 用 `IdentitiesOnly`、仓库别名和只读密钥 |
| 更新后仍是旧页面 | 查 SHA、构建时间、静态 root、缓存 | 拉错分支、未重建或入口指向旧目录 | 沿“提交 → 产物 → root → 缓存”核对 |
| 重启服务器后宕机 | 查 systemd/PM2/container 状态 | 未启用开机启动或环境未加载 | 修守护配置并再次重启演练 |

## 域名和证书迁移

迁移域名时分别处理：

1. 新域名 DNS 与证书；
2. 新域名正式入口；
3. 旧域名仍有效的证书；
4. HTTP 和 HTTPS 旧域名到新域名的 301；
5. 路径和查询参数保留；
6. 回调 URL、CORS、Cookie、OAuth 和站点地图更新；
7. 观察日志后再决定何时下线旧域名。

不能只给旧域名配置 HTTP 跳转：用户访问旧 HTTPS 时，TLS 握手发生在重定向之前。

## 上线后功能差异

本地成功、线上核心功能失败时，检查：

- 第三方站点是否根据数据中心 IP、地区、User-Agent、Referer 或 Cookie 返回不同结果；
- 服务器是否需要代理、IPv6、DNS 或可信 CA；
- 下载/上传是否受 Nginx、应用、磁盘、超时和流式传输限制；
- 外部 URL 抓取是否有 SSRF、重定向、内网地址和 DNS 重绑定风险；
- 浏览器直连与后端代理的 CORS、内容类型和安全策略是否不同；
- 真实客户端 IP 是否在反向代理后被正确识别。

不要为了让抓取或下载“能用”而关闭 SSRF、证书验证或访问控制。先复现差异，再设计最小、安全的兼容方案。
